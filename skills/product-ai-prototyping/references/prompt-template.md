# 可直接复制给 AI 的提示模板

## 模板 A：做产品改造原型

```text
请基于现有的【产品/页面/原型】进行扩展，不要完全重构。

一、母体信息
- 当前产品是：
- 当前必须保留的模块：
- 当前不能动的大框架：

二、本次改造目标
- 目标 1：
- 目标 2：
- 目标 3：

三、本次优先关注
- 入口设计
- 文件管理
- 角色差异
- 关键流程

四、本次先弱化/不处理
- 不做：
- 先弱化：

五、请先按这个顺序思考和输出
1. 场景划分
2. 角色划分
3. 核心对象
4. 关键流程
5. 功能调整
6. 逻辑调整
7. 页面结构建议
8. 高保真原型建议

六、要求
- 不同场景不要混在一起
- 不同角色看到的内容和操作项要区分
- 优先保留原有产品骨架
- 原型要体现状态和交互，不只是静态页面
```

## 模板 B：让 AI 帮你梳理产品方案

```text
请你作为产品经理协作助手，帮我把下面的需求整理成完整产品方案。

背景：
- 当前产品：
- 当前用户：
- 当前问题：

目标：
- 想新增的能力：
- 希望解决的问题：

请按以下结构输出：
1. 改造目标
2. 场景划分
3. 角色划分
4. 对象模型
5. 关键流程
6. 功能调整
7. 逻辑调整
8. 规则与约束
9. 系统边界
10. 一期建议

要求：
- 多写规则，少写空泛描述
- 把功能写成动作闭环
- 明确谁能看、谁能做、在什么条件下生效
```

## 模板 C：让 AI 判断哪些能力该放不同系统层

```text
下面这些能力是我在产品方案里设计的，请你帮我判断：

1. 哪些适合放在前台工作流
2. 哪些适合放在后台或配置中心
3. 为什么
4. 一期最小实现应该先做哪些

请按以下结构输出：
1. 前台承接能力
2. 后台承接能力
3. 系统边界原则
4. 一期最小实现建议

要求：
- 以高频使用、规则来源、全局配置、审计要求为判断标准
- 避免不同系统层重复建设
```

## 模板 D：让 AI 输出高保真原型前的澄清问题

```text
在你开始输出原型前，请先不要直接画页面。

请先基于我的需求，补齐最关键但缺失的信息，并只提最必要的问题。

优先从下面几个维度检查缺失项：
- 母体产品
- 改造目标
- 场景
- 角色
- 核心对象
- 关键流程
- 规则来源
- 交付形式

如果你认为信息已经足够，请直接给出你的理解框架，而不是重复我的原话。
```

## 模板 E：让 AI 复盘一次产品与原型协作过程

```text
请你回顾我们这次从需求到原型的全过程，帮我输出一份可复用的方法论。

请重点回答：
1. 我最开始是怎么提出需求的
2. 需求是如何一步步被拆解和收敛的
3. 哪些提问方式最有效
4. 哪些阶段最适合让 AI 输出什么
5. 如何把这套方法复用给其他产品经理

请输出为：
1. 方法论
2. SOP
3. 注意事项
4. 可复用模板
```

## 模板 F：让 AI 先做方案，再做原型

```text
请先不要直接输出页面。

先把我的需求整理成一个适合做原型的产品框架，再决定页面怎么画。

请先输出：
1. 你的需求理解
2. 关键业务目标
3. 场景划分
4. 角色划分
5. 对象模型
6. 关键状态
7. 关键流程
8. 风险点与待确认项

在以上内容确认后，再输出：
9. 页面结构
10. 交互规则
11. 高保真原型建议

要求：
- 不要跳过逻辑直接画页面
- 不要把功能点堆成清单，要整理成闭环
- 对有歧义的地方先给出判断框架
```
