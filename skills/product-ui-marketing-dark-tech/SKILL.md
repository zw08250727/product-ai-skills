# 产品 UI / 官网营销 / 深色科技风

使用本 Skill 时，将产品想法、落地页、控制台、数据看板或单文件 HTML 原型，优化成更可信、更适合 PC 端展示的产品体验。优先保证产品表达清楚，再提升视觉质感；页面要能说明产品做什么、引导用户视线，并在桌面端显得专业、完整、有商业化品质。

## 何时启用

- PC 优先的 SaaS、AI 产品、数据平台、开发者工具、B2B 系统、企业门户、产品官网或营销落地页。
- 需要高保真 HTML/CSS/React 原型，且视觉质量重要，尤其是桌面布局、首屏、产品演示区、数据看板、工作流图、价格区、功能矩阵、登录或 onboarding。
- 用户提到：科技感、AI 智能感、深色主题、暗黑高级感、产品设计、视觉升级、PC端、大屏、数据看板、工作流、产品官网、落地页、高保真、商业化、专业感、高级感。
- 用户要求生成新版本、另存文件、不要覆盖原版本。

## 核心原则

1. **PC 优先，而不是移动端拉宽**：先按 1280-1600px 桌面宽度设计，再向下适配。利用 PC 宽度展示产品上下文、对比、控制项、数据看板和工作流结构。
2. **产品优先，装饰其次**：主视觉应是可信的产品界面、看板、工作流、编辑器、地图、时间线或指挥中心。抽象光效只是氛围，不是产品本体。
3. **信息层级清楚**：首屏应快速回答：给谁用、解决什么问题、为什么重要、下一步做什么。
4. **一套视觉系统管到底**：统一背景、表面、文字、边框、强调色、状态色、图表色和阴影。避免 CSS、SVG、Canvas、JS 字符串里散落一次性色值。
5. **动效解释状态**：动画应表达实时数据、进度、扫描、连接、路由、生成或系统状态。避免高频闪烁抢夺阅读注意力。
6. **保护已有版本**：用户要求新版本、另存或不要覆盖时，先复制或创建同级新文件，再在新文件上修改。

## PC 产品布局模式

### 首屏 Hero

- 优先做沉浸式产品场景：文案覆盖或靠近一块全宽产品画面，并让常规桌面视口能看到下一段内容的一点线索。
- 当可以做出更丰富的产品场景时，避免落入普通“左文案 + 右卡片”的模板感布局。
- 产品视觉必须具体：数据看板、工作流泳道、数据表、AI 对话/智能体面板、地图、关系图、时间线、监控台或搭建器画布。
- 文案短而具体。除非附近有明确证据，否则少用“赋能未来”这类泛化口号。
- CTA 保持主次：一个主按钮，一个次按钮，不要让所有链接同等醒目。

### 产品演示界面

PC 端要做出可信的软件屏幕，而不是泛泛的装饰卡片：

- 顶部 chrome：标签页、搜索、状态、用户/组织信息、环境标识。
- 左侧或顶部导航：4-7 个可信模块。
- 主画布：指标、表格、工作流、图表、智能体任务队列、预览或时间线。
- 右侧洞察区：AI 建议、告警、下一步动作、实体档案或审计记录。
- 信息密度可以高于移动端，但间距仍要可读。

### 工作流 / 流程

- 当产品涉及阶段、自动化、管道、审批、智能体链路或数据流动时，使用工作流视觉。
- 优先使用扁平但有动态感的图形：泳道、节点、步骤轨道、路由线、队列卡片、依赖图、流程时间线。
- 目标是让用户读懂流程。如果工作流不可读，优先降低重 3D、过度模糊、深阴影和大幅 hover 位移。

### 数据看板

- 指标要像真实产品：包含标签、单位、趋势和时间语境。
- 图表组合要有意图：KPI 卡片、表格、折线/柱状图、漏斗、状态列表、时间线和告警。
- 空状态、加载态、成功态、错误态都应服从同一套视觉语言。
- 避免无意义的假数字；选择能支撑产品故事的指标。

## 视觉方向

### 深色科技配色

深色主题要有对比和克制：

| 角色 | 建议方向 |
| --- | --- |
| 背景 | 近黑海军蓝、炭黑或黑蓝，并带少量层次 |
| 表面 | `rgba(255,255,255,0.04-0.08)` 配低透明亮边 |
| 主文字 | 柔和白色，不要全局纯白 |
| 次文字 | 冷灰蓝，并保证对比度 |
| 主强调色 | 用于 CTA 和激活态的一种强色 |
| 辅助强调色 | 用于 AI、数据流、实时状态的第二种色 |
| 成功 / 警告 / 危险 | 语义色少量且一致使用 |

避免整个界面被单一色系统治。深色科技界面也需要中性色表面、文字层级、语义状态和少量受控强调色。

### 层次与质感

- 用间距、边框、透明度和细腻阴影建立层次。
- 玻璃拟态首先要可读：深色半透明面板、清晰亮边、克制 blur。
- 不要把 blur、glow、gradient、shadow、border、animation 全堆在同一个元素上，通常会显得噪。

### 视觉效果

适合 PC 产品的效果：

- 工作流或产品演示区域中的连接线、数据脉冲。
- AI/数据系统里的扫描线、光标轨迹、进度 shimmer、实时状态轻微闪烁。
- 按钮、标签页、卡片、表格行、图表的微交互。
- 极低透明度的网格、点阵、代码雨、星场或噪点纹理。

避免：

- 与产品无关的大面积装饰光斑。
- 密集文字后方持续全屏运动。
- 会造成布局跳动的 hover。
- 过度霓虹，导致对比下降或页面像海报而不像软件。

## 组件指导

- **导航**：高度稳定，激活态清楚，有产品/公司标识，右侧保留主要 CTA。
- **卡片**：用于重复项或产品模块，不要把每个 section 都包成卡片。
- **按钮**：主按钮强、次按钮安静，并包含 disabled/loading 状态。
- **表单/登录**：字段清晰、校验可见，安全和信任提示保持克制。
- **表格**：PC 表格要有用：状态标签、看起来可排序的表头、紧凑行高和行操作。
- **图表**：标注坐标、时间范围或单位。强调色要有意图，不要随机上色。
- **AI 面板**：展示 AI 正在做什么：计划、证据、置信度、建议、动作或执行轨迹。

## 文案与内容

- 只写面向产品和用户的文案，不写设计说明。避免“此区域展示了”“页面包含”等自我描述。
- 用具体产品结果、角色、对象和流程替代泛泛表达。
- 标题短，辅助文字易扫读。
- 根据产品语境使用可信术语：线索、工单、部署、告警、活动、模型、文档、订单、账户、任务、审批、审计、事故等。

## 响应式规则

- 桌面优先：条件允许时检查 1280px、1440px 和宽屏。
- 平板/移动端应简化列数、降低图表密度，并保证产品界面仍可用。
- 核心文字不要依赖 `vw` 缩放。
- 固定格式 UI 要稳定：看板、网格、工具栏、计数器、图块、图表和产品 chrome 都需要稳定尺寸。
- 长标签要有明确换行或截断策略，不能溢出容器。

## 实施流程

1. **检查当前页面/应用**：识别框架、已有 token、布局结构、组件习惯和视觉约束。
2. **确定产品故事**：先明确主要用户、产品对象、核心流程和证明点，再改视觉。
3. **建立或更新 token**：背景、表面、文字、边框、强调色、语义状态、图表色、focus ring、阴影/光效。
4. **搭建 PC 端构图**：导航、首屏产品视觉、演示/看板/工作流、功能证明、CTA 和页脚。
5. **谨慎加入动效**：只动画化状态、流动、进度、实时数据或交互反馈。
6. **扫除硬编码样式**：CSS 变量、inline style、SVG fill/stroke、Canvas/JS 样式字符串、图表配置、modal/toast 覆盖。
7. **视觉验证**：可行时检查桌面与移动截图，修复溢出、对比度、间距和布局跳动。

## 版本策略

- 用户说“新版本”“不要覆盖”“另存”等类似表达时，创建 `_v2`、`_dark`、`_pc` 或 `_visual` 之类的同级文件。
- 保留原文件用于对比和回滚。
- 最终回复中说明新文件路径和主要设计变化。

## 质量检查清单

- [ ] 首屏说明产品，并展示具体产品界面。
- [ ] 桌面布局充分利用宽度，而不是拉宽移动端卡片。
- [ ] CTA、导航、产品演示和支撑区块层级清楚。
- [ ] 视觉效果强化产品状态或数据流动。
- [ ] 深色表面、文字、边框和语义状态使用一致 token。
- [ ] 图表、表格、工作流节点和 AI 面板内容可信。
- [ ] 没有浅色旧样式或 inline style 破坏新视觉系统。
- [ ] 深色背景上的文字可读且不溢出。
- [ ] hover/loading/active/error 状态已设计，并且布局稳定。
- [ ] 用户要求保留原文件时，原文件没有被覆盖。

## 常见修复

- 页面像通用海报：加入真实产品界面，减少抽象装饰。
- 页面太平：增加表面层级、边框、细腻阴影和一层受控光效。
- 页面太吵：减少重复渐变、降低 blur、放慢动画、限制强调色数量。
- 工作流看不懂：扁平化图形、强化标签、降低 3D 效果、明确阶段顺序。
- PC 端像移动端放大：加入桌面导航、更丰富的产品演示、侧栏、表格/图表和宽屏网格。
- 深色模式发灰发糊：提高文字对比、拉开表面层次，并用语义强调色突出关键状态。
